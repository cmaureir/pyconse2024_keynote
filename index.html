4. Flow Graph

  • Bytecode instructions optimizations
  • Break specialized bytecode into a smaller sequence of bytecodes. (better to optimize)
  • For example: 'BINARY_OP_ADD_INT' gets transformed into _CHECK_VALIDITY_AND_SET_IP _GUARD_BOTH_INT _BINARY_OP_ADD_IT
  • Optimization on extra steps and reduce the bytecode list into a smaller-bytecode sequence

Tier 1 (warm code)

Fast execution and low resource consumption.

Tier 2 (hot code)

Fast execution and low resources consumption.
Avoid too much optimization and memory consumption.

Tier 3 (hotter code)

Fast execution and almost no resource consumption.
Larger than Tier 2, more optimizations, but more expensive.

  #include <Python.h>

  static char simple_docs[] = "hello(): prints hello :P\n";
  static PyObject* simple_hello(PyObject* self, PyObject* args){
      char *msg = "Hey PyConSE!!";
      return Py_BuildValue("s", msg);
  }
  static PyMethodDef simple_funcs[] = {
      {"hello",                   // ml_name
       (PyCFunction)simple_hello, // ml_meth
       METH_NOARGS,               // ml_flags
       simple_docs},              // ml_doc
      {NULL, NULL, NULL, NULL}
  };
  static struct PyModuleDef simplemodule = {
      PyModuleDef_HEAD_INIT, // m_base
      "simple",              // m_name
      NULL,                  // m_doc
      -1,                    // m_size
      simple_funcs           // m_methods
  };
  PyMODINIT_FUNC PyInit_simple(void){
      return PyModule_Create(&simplemodule);
  }
